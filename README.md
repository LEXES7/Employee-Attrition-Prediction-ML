# Employee Attrition Prediction
**Machine Learning Assignment | Group Project**

---

## Overview

This project applies supervised machine learning to predict employee attrition using the IBM HR Analytics dataset. Four classical ML algorithms are compared to identify which factors drive employees to leave and which model best predicts attrition risk.

**Problem:** Binary classification ‚Äî predict whether an employee will leave (Yes) or stay (No)  
**Dataset:** IBM HR Analytics Employee Attrition ‚Äî 1,470 employees, 35 features  
**Dataset URL:** https://www.kaggle.com/datasets/pavansubhasht/ibm-hr-analytics-attrition-dataset

## How to Run

### 1. Install dependencies
```bash
pip install catboost scikit-learn seaborn matplotlib pandas numpy
```

### 2. Download the dataset
Download from Kaggle and place the CSV at:
```
Dataset/WA_Fn-UseC_-HR-Employee-Attrition.csv
```

### 3. Run individual notebooks
Open any notebook in the `notebooks/` folder and run all cells top to bottom. Each notebook:
- Loads the dataset
- Preprocesses the data
- Trains its algorithm
- Evaluates with Accuracy, F1, Precision, Recall, ROC-AUC
- Exports metrics to `Performance_Metrics/`

### 4. Run the comparison notebook
After all 4 individual notebooks have been run, open `main.ipynb` and run all cells to generate the full comparison table and charts.



## Dataset Details

| Property | Value |
|---|---|
| Rows | 1,470 |
| Columns | 35 (31 after preprocessing) |
| Target | Attrition (Yes / No) |
| Class balance | 83.9% No ‚Äî 16.1% Yes |
| Missing values | None |

**Preprocessing applied:**
- Dropped 4 constant columns: `EmployeeCount`, `EmployeeNumber`, `Over18`, `StandardHours`
- Encoded target: Yes ‚Üí 1, No ‚Üí 0
- Handled class imbalance via `class_weight='balanced'` per algorithm
- 80/20 stratified train/test split (`random_state=42`)
